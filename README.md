# かなえの小論文添削教室 - デプロイと運用メモ

このリポジトリは静的サイトとして設計されています。GitHub Pages で公開する場合は、リポジトリ名を `your-username.github.io`（例: `viva7sk-glitch.github.io`）にするか、リポジトリの Pages 設定で `gh-pages` ブランチや `main` ブランチの `root` を指定してください。

運用上の注意:
- 現在のフォーム送信はローカル保存と「送信シミュレーション」です。実際にメールやサーバーに送る場合は以下のいずれかに差し替えてください:
  - Formspree / Formcarry / Getform などの受信サービスを利用する（フォームの `action` をそれらのエンドポイントに変更）
  - Netlify Forms を利用する（特定のフォーム属性の追加）
  - サーバーレス関数（AWS Lambda / Vercel / Netlify Functions）で受け取る
- 個人情報（氏名／メールアドレス）を扱うため、運用時はプライバシーポリシーと連絡先を明示してください。
- 添削ファイルのアップロードを受け付ける場合は、サーバー側のウィルススキャンとファイルサイズ上限を設けてください。

カスタマイズ案:
- 管理者用ダッシュボードを作成して、依頼の一覧管理・添削ファイルのアップロードを可能にする
- 支払い機能の導入（Stripe等）による有料サービス化
- 自動採点チェックリスト（文字数・構成チェック・表現の重複検出）をフロントで追加

使い方（テスト）:
1. ファイル一式をリポジトリにアップロードして GitHub Pages を有効化する。
2. index.html を開き、フォームに入力して「添削を依頼する」を押す。
3. 実装ではローカルStorageへ保存され、画面に送信完了メッセージが出ます。
